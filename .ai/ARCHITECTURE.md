# Architecture & Package Responsibilities

This document defines the high-level architecture and package-level responsibilities for the CMS MINT application.

## High-Level Overview
MINT is a web application designed to help CMS staff collaborate on model plans.
- **Frontend:** React 19 SPA (TypeScript), built with Vite, using Apollo Client for GraphQL and USWDS for styling.
- **Backend:** Go server providing REST and GraphQL (gqlgen) APIs.
- **Database:** Postgres, managed via Flyway migrations.
- **Authentication:** Okta integration with JWT tokens.
- **Background Jobs:** Faktory worker for asynchronous tasks.
- **Infrastructure:** Docker, AWS services.

## Backend Responsibilities (`pkg/`)

- **`appcontext`**: Setting and retrieving per-request information (trace IDs, etc.).
- **`apperrors`**: Custom error types used to inform HTTP response codes (e.g., `UnauthorizedError`).
- **`cedar`**: Interaction with the CEDAR API and translation to internal models.
- **`graph`**: GraphQL schema (`schema.graphql`), generated code, and handwritten resolvers (`resolvers/`).
- **`handlers`**: HTTP handlers that parse requests, offload logic to services, and format responses.
- **`models`**: Shared data structures used across the application.
- **`services`**: The entry point for business logic; orchestrates auth, database, and third-party APIs.
- **`storage`**: All database interaction and SQL code. No SQL should exist outside this package.
- **`testhelpers`**: Shared testing utilities across packages.

## Frontend Responsibilities (`src/`)

- **`app`**: Application entry point, global styling, and high-level routing.
- **`features`**: Page-level components or collections of pages representing a unified responsibility.
- **`components`**: Individual, reusable UI components (colocated with CSS and tests).
- **`gql`**: GraphQL operations (`operations/`) and autogenerated types (`generated/`).
- **`hooks`**: Custom React hooks for shared logic.
- **`wrappers`**: Higher-order components for routing, authentication state, and feature flags.
- **`i18n`**: Centralized text management for internationalization.

## Data Flow Paradigm
1. **Frontend** triggers a GraphQL operation via Apollo.
2. **GraphQL Resolver** (`pkg/graph`) receives the request.
3. **Resolver** calls a **Service** (`pkg/services`) for business logic and auth checks.
4. **Service** calls **Storage** (`pkg/storage`) for database operations.
5. **Storage** returns **Models** (`pkg/models`) back up the chain.
